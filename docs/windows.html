Windows Setup

Get OSMOzzz running on Windows in under 5 minutes.

Note: On Windows, macOS-specific sources (iMessage, Safari, Apple Notes, Apple Calendar) are not available. Chrome history, Files, Terminal and cloud connectors work normally.
Step 1

Download

Click the button below to download the latest Windows release.

Download osmozzz-windows.zip

Once downloaded, unzip the file. You'll get:

# Contents of osmozzz-windows.zip
osmozzz.exe
onnxruntime.dll
Step 2

Install

Open PowerShell as Administrator, navigate to the unzipped folder and run:

# Go to the unzipped folder
cd $env:USERPROFILE\Downloads\osmozzz-windows

# Install (copies binary + registers as MCP server)
.\osmozzz.exe install
Windows Defender warning? Click "More info" → "Run anyway". OSMOzzz is open source — you can verify the code on GitHub.

This will copy osmozzz.exe to %USERPROFILE%\.cargo\bin\ and register it as a Claude Desktop MCP server.

Step 3

Start the daemon

The daemon indexes your data in the background. Open PowerShell and run:

osmozzz daemon

Open the local dashboard in your browser:

http://localhost:7523

The dashboard shows all indexed sources and lets you configure cloud connectors (Notion, GitHub, Slack…).

To start the daemon automatically on Windows startup, add it to Task Scheduler or add it to your startup folder.
Step 4

Connect Claude Desktop

osmozzz install automatically registers OSMOzzz as an MCP server in Claude Desktop. Just restart Claude Desktop and it's done.

To verify: Claude Desktop → Settings → Developer → you should see osmozzz listed as an MCP server.

Don't have Claude Desktop yet? Download it here →
Step 5

Verify it works

Open Claude Desktop and ask:

"Search my files for anything about project X"

Or check the status in PowerShell:

osmozzz status
✓ OSMOzzz is ready. Chrome history, Files (Desktop/Documents/Downloads) and Terminal history are being indexed automatically.
